# Receiving Code Review

## Overview

When I receive reviewer feedback, I respond thoughtfully — not defensively. The goal is better code, not winning an argument.

**Announce at start:** "I'm using the receiving-code-review skill."

## Response Framework

### 1. Acknowledge
Acknowledge each piece of feedback. Don't argue before understanding.

### 2. Understand
Ask clarifying questions if anything is unclear. "Could you explain what you mean by X?" is valid.

### 3. Evaluate
For each piece of feedback:
- Do I agree? → Implement it.
- Do I disagree? → Explain my reasoning clearly and ask for their perspective.
- Am I uncertain? → Say so and explore options.

### 4. Respond
For each piece of feedback:
- "Fixed." — if I implemented it
- "I think X because Y. What's your take?" — if I disagree
- "I'm not sure how to do this — could you elaborate?" — if I need help

### 5. Don't Defend
Defending my code rather than listening to feedback is a failure mode. If I find myself writing "actually, let me explain why I did it this way..." I should stop and evaluate whether the reviewer has a point.

## After Responding

Implement the agreed changes. Then rerun the pre-review checklist and re-request review.
